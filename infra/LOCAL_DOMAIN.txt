Локальный аналог регистрации домена и делегирования
==================================================

В облаке: регистратор выдаёт домен, у DNS-провайдера создаёте зону и A-запись
на IP балансировщика, у регистратора указываете NS-серверы провайдера
(делегирование зоны — см. https://ru.wikipedia.org/wiki/%D0%94%D0%B5%D0%BB%D0%B5%D0%B3%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D0%B5_%D0%B4%D0%BE%D0%BC%D0%B5%D0%BD%D0%B0 ).

Локально это заменяется так:

1) «Регистратор» — вы сами выбираете имя из зарезервированных для документации
   имён (например devops.example, example.local). Реальный аккаунт у Namecheap
   или Reg.ru для учебного проекта не обязателен, если вы работаете только
   на своей машине.

2) «A-запись на балансировщик» — сервис dns в docker-compose (CoreDNS) отдаёт
   для этих имён адрес 127.0.0.1, где слушает контейнер lb (порт 80 проброшен
   на хост). Это аналог A-записи на публичный IP ALB.

3) «Делегирование NS» — в проде на домен вешают NS облачного DNS. Локально
   достаточно направить запросы на локальный резолвер: контейнер dns слушает
   UDP/TCP 5353 на 127.0.0.1.

Проверка записи (после make up):

  dig @127.0.0.1 -p 5353 devops.example A +short
  dig @127.0.0.1 -p 5353 example.local A +short

Ожидается ответ 127.0.0.1.

Проверка приложения по имени:

  curl -s -H "Host: devops.example" http://127.0.0.1/
  curl -s --resolve devops.example:80:127.0.0.1 http://devops.example/

Чтобы браузер открывал http://devops.example/ без --resolve, добавьте в
/etc/hosts строку:

  127.0.0.1 devops.example www.devops.example example.local www.example.local

либо настройте систему использовать DNS 127.0.0.1:5353 (зависит от ОС).
